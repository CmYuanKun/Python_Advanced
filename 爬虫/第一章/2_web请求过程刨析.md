# web请求过程：
1. 服务器渲染：在服务器那边直接把数据和html整合在一起，统一返回给浏览器在页面源代码中能看到数据。
2. 客户端渲染：第一次请求只有一个html骨架，第二次请求可以拿到数据，进行数据显示。在页面源代码中，看不到数据
3. 熟练使用浏览器抓包工具

# HTTP协议：
## 请求
1. 请求行 -> 请求方式(get/post)请求url地址 协议
2. 请求头 -> 放一些服务器使用的附加信息
3. 请求体 -> 一般放一些请求参数

## 响应
1. 状态行 -> 协议 状态码
2. 响应头 -> 放一些客户端要使用的一些附加信息
3. 响应体 -> 服务器返回的真正客户端要用的内容(html,json)等
